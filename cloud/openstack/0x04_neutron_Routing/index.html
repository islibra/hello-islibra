    0x04_neutron_Routing¶

    不同VLAN之间通过router连通, 路由上配置多个VLAN的网关地址。

    KVM实现 - L3 agent¶

    在控制节点或网络节点运行虚拟router。

    配置文件：/etc/neutron/l3_agent.ini

    • linux bridge: interface_driver = neutron.agent.linux.interface.BridgeInterfaceDriver
    • open vswitch: interface_driver = neutron.agent.linux.interface.OVSInterfaceDriver

    查看agent列表：neutron agent-list

    1. 通过Web GUI创建router

      1. 增加interface, 选择vlan的subnet

    Linux Bridge实现原理¶

    在bridge上分别增加veth pair连接TAP设备和router namespace 中的interface。

    如果不使用namespace, 多个租户创建的路由条目只能记录到控制节点的路由表中, 会产生重叠

    • 查看router列表：neutron router-list

    • 查看namespace:

      1
2
      $ ip netns
qrouter-{routerid}

    • 查看router中的veth interface配置:

      1
2
      $ ip netns exec qrouter-{routerid} ip a
qr-xxx

    • 查看router中的路由表：ip netns exec qrouter-{routrtid} route

    访问外网¶

    1. 配置网络类型和网卡映射: /etc/neutron/plugins/ml2/ml2_conf.ini

      • flat

      • vlan

    2. 重启neutron服务

    3. 通过Web GUI创建外网

      创建外网 ext_net - 每天5分钟玩转 OpenStack（104）

      1. 勾选 外部网络
      2. 填写 Gateway IP
      3. 禁用DHCP

    4. 查看linux bridge状态: brctl show, 已创建brqXXX并挂载eth2

    5. 设置router的Gateway为External Network, 自动增加interface, IP来自外部网络

    6. 查看网桥已挂载router的tap

    7. namespace中新增qgxxx

    数据包从qgxxx发出的时候会进行SNAT为router的interface地址10.10.10.2

    1. 查看SNAT规则: ip netns exec qrouter-xxx iptables -t nat -S

    2. tcpdump验证SNAT

    floating IP¶

    1. 通过webUI创建Floating IPs

      创建 floating IP - 每天5分钟玩转 OpenStack（106）

    2. 在ext_net中Allocate IP, 并Associate到VM

    3. 在namespace中查看router的interface qgxxx添加floating IP

    4. 在namespace中查看iptables中的NAT规则

      1. 如果目的地址是10.10.10.3, DNAT为172.16.101.3
      2. 如果源地址是172.16.101.3, SNAT为10.10.10.3

    5. tcpdump验证

    在 router 的 interface 上设置 floating IP, 并通过 iptables 配置 NAT

    访问外网¶

    1. 将eth2挂到br-ex上

    2. 配置外部网络与网桥的映射: /etc/neutron/plugins/ml2/ml2_conf.ini

      • flat

      • vlan

    3. 重启neutron

    4. 通过webUI创建ext_net

      创建 OVS 外部网络 ext_net - 每天5分钟玩转 OpenStack（144）

      1. 勾选 外部网络
      2. 填写 Gateway IP
      3. 禁用DHCP

    5. 执行ovs-vsctl show查看已创建patch port连接br-int和br-ex

    6. 将router Set Gateway为ext_net

      • br-ex自动增加port qgxxx

      • namespace中的router自动增加interface qgxxx, IP来自于ext_net

    7. namespace路由表中默认网关指向qgxxx
